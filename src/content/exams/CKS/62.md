# Advanced Pod Security

## Pod Security Standards (PSS)
Replaced PodSecurityPolicies (PSP). Defined in 3 levels:
1. **Privileged**: Unrestricted (Admin level).
2. **Baseline**: Minimally restrictive (Default).
3. **Restricted**: Heavily restricted (Best practice).

## Pod Security Admission (PSA)
The built-in admission controller to enforce PSS via Namespace labels.

# Task 1: Enforce PSS
Label the `dev` namespace to enforce the `restricted` pod security standard.

```bash
kubectl create ns dev 2>/dev/null || true
kubectl label ns dev pod-security.kubernetes.io/enforce=restricted --overwrite
```

```setup
kubectl create ns dev 2>/dev/null || true
kubectl label ns dev pod-security.kubernetes.io/enforce- 2>/dev/null || true
```

```verify
kubectl get ns dev -o jsonpath='{.metadata.labels}' | grep "pod-security.kubernetes.io/enforce" | grep "restricted"
```
