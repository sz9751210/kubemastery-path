# Mastering YAML

Kubernetes is **Declarative**. syntax matters.

## The 4 Pillars
1. **apiVersion**: \`v1\`, \`apps/v1\`.
2. **kind**: \`Pod\`, \`Service\`.
3. **metadata**: \`name\`, \`labels\`.
4. **spec**: The desired state.

\`\`\`yaml
apiVersion: v1
kind: Pod
metadata:
  name: demo
spec:
  containers:
  - name: nginx
    image: nginx
\`\`\`
