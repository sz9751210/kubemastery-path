# Mastering YAML

Kubernetes is **Declarative**. syntax matters.

## The 4 Pillars
1. **apiVersion**: \`v1\`, \`apps/v1\`.
2. **kind**: \`Pod\`, \`Service\`.
3. **metadata**: \`name\`, \`labels\`.
4. **spec**: The desired state.

# Task 1: Create from YAML
Create a pod using the following YAML. Save it to `pod.yaml` and apply it.

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: demo
spec:
  containers:
  - name: nginx
    image: nginx
```

```setup
# Clean up if exists
kubectl delete pod demo --force --grace-period=0 2>/dev/null || true
```

```verify
kubectl get pod demo --no-headers | grep Running
```
