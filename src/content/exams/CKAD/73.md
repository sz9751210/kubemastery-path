# Services & Networking Design

## Network Policies
Isolate your applications.
- **Ingress**: Incoming traffic.
- **Egress**: Outgoing traffic.

## Service Discovery
- **DNS**: Every Service gets a DNS name.
- **Debugging DNS**: Use \`nslookup\` or \`dig\` from within a pod.

# Task 1: Test DNS
Verify that you can resolve `kubernetes.default`.

```bash
kubectl run test-dns --rm -it --image=busybox -- nslookup kubernetes.default
```

```setup
# Clean up if exists
kubectl delete pod test-dns --force --grace-period=0 2>/dev/null || true
```

```verify
# Simply checking if the cluster DNS service exists
kubectl get svc kube-dns -n kube-system > /dev/null
```
