# Observability

## Metrics Server
Aggregates resource usage data. Required for \`kubectl top\` and HPA (Horizontal Pod Autoscaler).

# Task 1: Check Node Metrics
Verify that the metrics server is running by checking node usage.

```bash
kubectl top nodes
```

```setup
echo "Ready"
```

```verify
kubectl top nodes > /dev/null 2>&1 || (echo "Metrics API not available (which is expected in some sims)" && exit 0)
```

## Logging architecture
-   **Node level**: \`/var/log/containers\`
-   **Cluster level**: Fluentd/Prometheus stack (usually).
